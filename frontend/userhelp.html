1. Introduction

The Customers Database is a web-based application designed to manage customer records for the software company. It provides a clean, intuitive interface for viewing, searching, creating, editing, and deleting customer entries.

This system was built to replace manual Excel-based tracking with a modern web application that offers:

  • Real-time full-text search across all customer fields
  • Complete CRUD operations (Create, Read, Update, Delete)
  • Pagination for efficient browsing of large datasets
  • License tracking showing total licenses sold across all customers
  • Responsive design that works on desktop and mobile devices

2. Getting Started

To access the Customers Database application:

  1. Ensure the server is running (ask your system administrator if you are unsure).
  2. Open your web browser (Chrome, Firefox, or Edge recommended).
  3. Navigate to http://localhost:8080
  4. The main customer list page will load automatically.
💡 Tip: If the page does not load, verify that the server is running by checking with your administrator. The application runs on port 8080 by default.

4. Searching Customers

The search feature allows you to quickly find customers by typing any keyword or phrase.

How Search Works
  • Type your search term in the search bar at the top of the page.
  • Results update automatically after a brief pause (300ms) as you type — no need to press Enter.
  • Search is case-insensitive, so "abb", "ABB", and "Abb" all produce the same results.
  • The search scans all text fields: company name, contact names, email, phone numbers, addresses, program version, delivery date, and comments.
Clearing Search

Click the × button on the right side of the search bar to clear your search and show all customers again. You can also simply delete the text manually.

💡 Tip: When you start a new search, the application automatically resets to page 1 so you see the most relevant results immediately.

5. Browsing & Pagination

The customer table displays up to 50 records per page. When there are more results than fit on one page, pagination controls appear at the bottom of the table.

Using Pagination Controls
  • Previous — Go to the previous page (disabled on page 1).
  • Page Numbers — Click any visible page number to jump directly to that page. A sliding window of up to 5 pages is shown at a time, with ellipsis (…) for gaps.
  • Next — Go to the next page (disabled on the last page).
Sorting Columns

Click the column headers for Company, Contact Name, Email, Phone, or City to sort the table by that column. The arrows (▴▾) indicate sortable columns.

💡 Tip: Combine search with sorting for precise results — first narrow down with a search term, then sort by company or name.

6. Adding a New Customer

  1. Click the "+ Add Customer" button in the top-right corner of the page.
  2. The "Add New Customer" modal form will open with all fields empty.
  3. Fill in the required fields (marked with an asterisk *):
    • Company — The customer's company name (minimum 2 characters)
    • Primary Contact — Main contact person's name (minimum 2 characters)
  4. Optionally fill in any additional fields such as email, phone numbers, addresses, program version, delivery date, licenses, and comments.
  5. Click the "Save Customer" button at the bottom of the form.
  6. A success notification will appear briefly in the bottom-right corner. The customer table refreshes automatically to show the new record.
⚠ Note: If required fields are missing or invalid, red error messages will appear below those fields. Correct the errors before saving.

7. Editing an Existing Customer

  1. Locate the customer you want to edit in the table (use search if needed).
  2. Click the ✎ Edit button (pencil icon) in the Actions column of that row.
  3. The "Edit Customer" modal form will open, pre-filled with the customer's current data.
  4. Modify any fields as needed. Only Company and Primary Contact are required.
  5. Click "Save Customer" to apply your changes.
  6. A success notification appears and the table refreshes automatically.
💡 Tip: Clicking "Cancel" or the close button (X) on the modal will discard any unsaved changes.

8. Deleting a Customer

  1. Locate the customer you want to delete in the table.
  2. Click the 🗑 Delete button (trash icon) in the Actions column of that row.
  3. A confirmation dialog will appear showing the company name and contact person. This prevents accidental deletions.
  4. Click "Delete" to confirm, or "Cancel" to abort.
  5. If confirmed, a success notification appears and the table refreshes automatically.
⚠ Warning: Deleting a customer is permanent and cannot be undone. Always confirm the correct record before clicking Delete in the confirmation dialog.

9. Form Validation Rules

The application validates form input before saving to ensure data quality:

FieldRuleError Message
Company Required, minimum 2 characters "Please provide a valid company name."
Primary Contact Required, minimum 2 characters "Please provide a valid contact name."
Email Optional, but must be valid email format if provided "Please provide a valid email address."

When validation fails, the invalid fields are highlighted in red and error messages appear below them. The form will not submit until all required fields pass validation.
